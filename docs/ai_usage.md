## Where AI Assisted

### Architecture and design

- Initial discussion of service boundaries: where to draw the line between controllers,
  models, and service objects
- Review of the cache-aside pattern implementation and the three-path flow
  (Redis hit / DB hit / Gemini call)
- Discussion of AES-256-GCM vs CBC and the implications of the authentication tag
- Shaping the `MockEmailService` interface to mirror the future `GraphEmailService` contract

### Implementation

- Generating boilerplate for service classes once the design was agreed upon
- Writing factory definitions in FactoryBot and identifying that `Faker::Name.full_name`
  does not exist in Faker 3.8 (caught during test runs)
- Identifying the correct byte layout of the GCM ciphertext (`iv + auth_tag + encrypted`)
  for the tampered-ciphertext test case
- Writing RSpec examples for request specs, model specs, and service specs

### Debugging

- Diagnosing a `SyntaxError` caused by writing `trait :admin { role { "admin" } }` — Ruby
  parses the block as a hash argument, not a block; fixed to `do...end` form
- Identifying that `ciphertext.reverse` on a Base64 string produces invalid Base64
  (raising `ArgumentError`) before OpenSSL gets to verify the auth tag

### Documentation

- Drafting all four documents in `docs/` and the `README.md`
- Structuring the decisions document and the architecture diagrams

## Review Process

All AI-generated code was reviewed before acceptance:

- Each service and controller was read in full after generation to verify correctness
- The RSpec suite was run after every significant change; no code was considered done until
  tests passed
- `bundle exec rubocop` was run and all 56 initial offenses were corrected
- `bundle exec brakeman -q` returned 0 warnings
- `bundle exec bundler-audit check --update` found no vulnerable dependencies
- Manual API verification was performed with curl against the running Docker stack

## Honest Summary

AI significantly accelerated the implementation. Boilerplate, factory setup, spec structure,
and documentation that would have taken hours to write were produced in minutes.

The areas where AI was most valuable were the ones with clear specifications and known
patterns: service classes with defined interfaces, RSpec examples for well-understood
behavior, and documentation from an agreed architecture.

The areas that required the most human judgment were the design decisions — what to encrypt,
how to invalidate the cache, when to use a service vs. a model method, and what to skip
entirely. Those decisions were not generated by AI; they were discussed with AI as a
sounding board and then made by the engineer.
